# Прототип сцены хоррора
Данная сцена создана на основе тестового задания. Сцена реализует следующие механики и задачи: взаимодействие игрока с объектами и NPC; преследование игрока NPC; постоянная частота кадров; VHS-фильтр.

## Описание тестового задания
Сделать игру на 1-2 минуты. 
1. Игрок появляется в приятном и уютном кафе.
2. К нему подходит NPC клиент.
3. Игрок берет пустой стаканчик - ставит в кофемашину - стаканчик наполняется (можете только звуком, как вариант) - игрок берет крышку и соединяет её со стаканчиком - после чего берет кофе и кидает его в NPC.
4. После нужно сделать анимацию погоню, это может быть монстр, который ворвётся в кафе, либо это может быть клиент, которому не понравился кофе и он побежит за игроком, либо что-то другое. Но нужно, чтобы за нами кто-то побежал, экран начал двигаться, был страшный звук погони.
5. Также необходимо добавить вверху экрана счётчик ФПС (игра работает стабильно без скачков от 60 FPS на средних ПК).
Уютный, тёплый жёлтый свет, VHS-фильтр. Вокруг здания темно - ночь, но светло внутри - желтый уютный свет: игрок чувствует внутри уют, снаружи — угрозу.

## Описание проекта
Данный проект демонстрирует взаимодействие с объектами, работу с NavMesh и AI, управление персонажем от первого лица, UI и UX подсказки, оптимизацию и архитерктуру кода. Взаимодействие с объектами реализовано с помощью интерфейсов, действия NPC осуществляются с помощью NavMeshAgent, также в проекте используется постпроцессинг и "Graphy – Ultimate Stats Monitor" для отображения FPS и производительности.
